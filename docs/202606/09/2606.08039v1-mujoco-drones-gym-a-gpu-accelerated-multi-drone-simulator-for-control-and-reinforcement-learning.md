## 摘要
机器人模拟器是空中机器人现代研究的基石，既可作为新控制算法开发的工具，也可作为训练强化学习策略的数据源。然而，现有的四旋翼学习环境通常在物理保真度、多智能体支持以及现代深度强化学习流程所需的吞吐量之间面临权衡。在本文中，我们介绍了MuJoCo-Drones-Gym，这是一个基于MuJoCo物理引擎构建的开源、兼容Gymnasium的多无人机环境。MuJoCo-Drones-Gym支持任意数量的Bitcraze Crazyflie 2.x纳米四旋翼，并提供一个模块化API，用于选择：(i)物理模型（刚体MuJoCo、显式Python动力学，或地面效应、桨叶阻力、无人机间下洗流的任意子集）；(ii)动作接口（每电机转速、集体归一化推力、速度设定点或PID航点指令）；以及(iii)观测空间（运动状态向量、RGB/深度/分割相机或邻域邻接信息）。PettingZoo ParallelEnv封装支持即插即用的多智能体强化学习，而一套包含悬停、速度追踪、多无人机悬停、航点导航、编队飞行、门道竞速以及通用多智能体模板的七个任务环境则展示了该接口的广度。我们描述了环境设计、底层物理和四旋翼动力学，并通过与密切相关的gym-pybullet-drones项目类似的控制和学习示例来展示其使用，同时利用MuJoCo改进的接触处理、渲染和并行化能力。

## Abstract
Robotic simulators are a cornerstone of modern research in aerial robotics, serving both as a vehicle for the development of new control algorithms and as the data source for training reinforcement learning (RL) policies. Yet, existing quadcopter learning environments often face a trade-off between physical fidelity, multi-agent support, and the throughput required by modern deep RL pipelines. In this paper, we present MuJoCo-Drones-Gym, an open-source Gymnasium-compatible multi-drone environment built on top of the MuJoCo physics engine. MuJoCo-Drones-Gym supports an arbitrary number of Bitcraze Crazyflie 2.x nano-quadcopters and exposes a modular API for selecting (i)~the physics model (rigid-body MuJoCo, explicit Python dynamics, or any subset of ground effect, blade drag, and inter-drone downwash), (ii)~the action interface (per-motor RPMs, collective normalized thrust, velocity setpoints, or PID waypoint commands), and (iii)~the observation space (kinematic state vectors, RGB / depth / segmentation cameras, or neighbourhood adjacency information). A PettingZoo ParallelEnv wrapper enables drop-in multi-agent reinforcement learning, while a suite of seven task environments, hover, velocity tracking, multi-drone hover, waypoint navigation, formation flight, gate racing, and a generic multi-agent template, demonstrates the breadth of the interface. We describe the environment design, the underlying physics and quadcopter dynamics, and illustrate its use through control and learning examples that mirror those of the closely related gym-pybullet-drones project, while taking advantage of MuJoCo's improved contact handling, rendering, and parallelizability.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：基于 MuJoCo 物理引擎构建 Gymnasium 兼容的多无人机环境，支持任意数量 Crazyflie 2.x 四旋翼，提供模块化 API，可切换物理模型、动作接口和观测空间。同时集成 PettingZoo ParallelEnv 以支持多智能体强化学习。
- **关键技术细节**：
  - **四旋翼动力学（3.1节）**：状态向量为 `x = [p, q, v, ω]ᵀ ∈ R¹³`。电机推力与力矩公式：`F_i = k_f n_i²`，`τ_z,i = (-1)^(m_i) k_m n_i²`（交替符号产生偏航控制）。对于 X 构型，滚转与俯仰力矩：`τ_x = L/√2 (F0+F1-F2-F3)`，`τ_y = L/√2 (-F0+F1+F2-F3)`。集总推力 `T = ΣF_i`。
  - **空气动力学效应**（可独立开关）：
    - 地面效应：`ΔF_gnd_z = C_gnd Σ(k_f n_i²) * (r_p/(4h))²`，`h = max(p_z, h_clip)`，防止倒置时误用。
    - 桨叶阻力：体坐标系速度比例阻力，`F_drag = -diag(c_xy,c_xy,c_z) Σ(2πn_i/60) v_body`，再转换到世界坐标系。
    - 下洗流：当两无人机垂直重叠且横向距离<10m时，下方无人机受到额外向下力：`ΔF_dw_z = -α exp(-½(Δ_xy/β)²)`，参数由 Crazyflie 系数确定。

## 6. 主要结论与发现

1. MuJoCo-Drones-Gym 成功将 gym-pybullet-drones 的设计迁移到 MuJoCo 引擎，利用了后者更优的接触处理、渲染和并行能力。
2. 提供了更丰富的功能集：七种任务、六种物理模式、五种动作类型、三种观测类型、三种无人机模型、风场/障碍物/课程/域随机化包装器。
3. 通过 PettingZoo ParallelEnv 直接支持多智能体 RL，降低了 MARL 研究的使用门槛。
4. 通过 MJX/JAX 后端实现了 GPU 向量化，可同时运行数千个环境，适配 JAX 原生训练管线。
5. 物理模型（地面效应、阻力、下洗流）与 gry-pybullet-drones 公式一致，保证了策略的可移植性。
6. 提供的控制器（PID 和 SE(3) 几何控制器）作为基线有效，特别是几何控制器在激进轨迹上优于纯位置 PID。
7. 该平台有望成为多无人机控制和强化学习研究的新基准，未来可扩展至 sim-to-real 和视觉 RL。

## 7. 优点

- **设计理念先进**：基于现代 MuJoCo 引擎，支持 GPU 向量化（MJX+JAX），兼容最新 Gymnasium/PettingZoo API，具有前瞻性和可扩展性。
- **模块化与灵活性**：物理模式、动作类型、观测类型均可独立选择，包装器可组合，适用于从经典控制到深度 RL 的广泛研究。
- **功能全面**：提供丰富任务环境、控制器、空气动力学效应、风场、障碍物、课程、域随机化，覆盖实际研究中的多数需求。
- **易用性与可移植性**：纯 Python、pip 可安装；与 gym-pybullet-drones 的 API 和动力学接口相近，方便现有代码迁移；提供完整测试套件和示例。
- **开源与协作**：代码公开，鼓励贡献，资产（网格）内建，无需外部仓库。

## 8. 不足与局限

- **实验不充分**：缺乏与 gym-pybullet-drones 的定量性能/精度对比；无大规模多任务 RL 收敛曲线和消融实验；无 sim-to-real 实际硬件验证。
- **MJX 后端功能受限**：目前仅支持刚性 MuJoCo 物理，未实现地面效应、阻力、下洗流等空气动力学，且不支持视觉观测和多机交互，限制了全任务覆盖。
- **缺少资源消耗报告**：未提供训练效率、GPU 利用率、吞吐量等实际数据，无法判断其相对于旧框架的加速比。
- **未评估 sim-to-real 效果**：虽然准备了域随机化包装器，但没有在真实 Crazyflie 上验证迁移性能，实际落地有效性未知。
- **多智能体场景有限**：仅提供了一个通用模板（MultiAgentAviary），没有内置协作/竞争复杂奖励函数或通信机制。
